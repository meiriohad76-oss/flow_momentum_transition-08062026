Product concept

What the agent does

The agent surfaces unusual trading activity — abnormal volume, large off-exchange / block prints, and directional pressure — and classifies each name into a rule-based tier. It is explicitly an evidence tool, not a black box: it never collapses signals into a single opaque score, and it never infers direction from price.

Three independent indicators

Every signal is read on A (universe percentile), B (historical z-score vs the name's own history), and C (raw metric). They are shown separately and never averaged.

Rule-based tier, not a model

Tiers A / B / C / D come from explicit threshold gates that the user can read and audit. "Explain this tier" shows the exact rules that passed or failed.

Honest data lanes

Each data source is a "lane" with one of 9 explicit states. A tier is never emitted on incomplete data — degraded coverage caps or suppresses the tier instead of guessing.

Core principle to preserve in the build: direction is derived from signed order flow (Lee-Ready signing on prints), never from price change. Price is only ever used as corroboration. Keep this separation everywhere — it is the product's credibility.

App shell

Persistent chrome

Top bar — TopBar

  • Brand / home button — returns to Landing.
  • Mode tabs — Single Ticker · Portfolio · Scan · Alerts. Alerts carries a count badge (alertCount = "needs attention").
  • Ticker search — autocompletes from the ticker list; submitting a known symbol routes to Single Ticker.
  • Watchlist pill — opens the watchlist drawer; shows saved count.
  • Theme toggle — dark ⇄ light (writes the theme tweak).
  • Sync indicator — "Live · synced HH:MM:SS ET", or a spinning "Revalidating lanes…" pill during a revalidation event.

Market-Regime banner — RegimeBanner

Shown on every mode except Landing. Communicates macro context that shades signal interpretation.

  • Regime badge — Risk-On / Neutral / Risk-Off / Crisis, color-keyed.
  • VIX, yield-curve spread, Fed Funds stats.
  • Interpretation line — plain-English caution (e.g. risk-off ⇒ "treat bullish signals cautiously").
Regime never changes a tier. It only frames the BLUF "Limitations" and the macro corroboration flag.

Global behaviors

Cross-cutting interactions

Lane revalidation

Any "refresh lane" action dispatches a uta:revalidate event. The shell shows a top progress bar + "Revalidating lanes…" pill for ~1.9s, then stamps a fresh sync time. Use this same pattern for real refetches.

Persistence (localStorage)

Watchlist, review state (reviewed/ignored), alert rules, saved scans, scan-results view preference, and tweaks all persist locally. Keys are namespaced uta_*_v1. In production these become user-scoped server state.

Theming & density

Theme (dark/light), accent, density (compact/regular/comfy), and typeface are driven by data-attributes on <html> and CSS variables. No component hard-codes a color — all reference tokens.

Cycle model

The agent runs on a 5-minute cycle. "Δ cycle" everywhere compares the current cycle to the prior one. Tier changes, deltas, and the cycle-history timeline all key off this cadence.
