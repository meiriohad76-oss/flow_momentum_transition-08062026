Part 04 · Build Tickets

From spec to backlog

Ten epics broken into implementation tickets, each with acceptance criteria and dependencies, sequenced into four phases. Points are relative (1 ≈ half-day, 8 ≈ a week). Build the foundation and data layer first — every mode depends on them — then the shared detail view, then the four modes in parallel, then cross-cutting polish.

10 epics 4 phases ~48 tickets

Delivery plan

Four phases
P0

Foundations

Design system, app shell, data layer & tier engine. Nothing renders correctly without these.

P1

Detail view

The shared ticker detail — BLUF, evidence, corroboration, drawers. The product's core surface.

P2

Modes

Single Ticker, Portfolio, Scan, Alerts — buildable in parallel once the detail view exists.

P3

Cross-cutting

Watchlist, persistence, responsive, accessibility, motion & polish.

Phase 0 · Foundations

E1

Design system & tokens

Phase 0
DS-1Token layer3 pts

Implement the full CSS variable system: color, ink scale, surfaces, direction/regime keys, plus density-driven spacing/radii/type, for both themes.

  • All tokens from styles.css present for dark + light
  • data-theme and data-density on root switch every value
  • No component references a literal color or px outside the token set
DS-2Component primitives5 pts

Build buttons, tier badge (4 tiers × 3 sizes + word), direction tags, band tags, indicator chips (A/B/C + N/A), pills, key-value rows, cards, and the collapsible evidence-card shell.

  • Matches the Design System demos pixel-for-pixel in both themes
  • Tier badge encodes A solid / B soft / C ring / D dashed correctly
  • Evidence card head: icon, title/sub, headline metric, rotating chevron, bordered body; collapse persists per instance
Depends: DS-1
DS-3Icon system2 pts

Single stroked icon set behind one <Icon name size fill /> component; multi-path glyphs supported.

  • All ~45 glyphs render at 1.7 stroke on a 24px grid
  • Optional fill flag for solid states (e.g. saved bookmark)
DS-4Mini-chart primitives5 pts

Sparkline (line + area, optional zero baseline), VolBars (today vs ghost baseline, hot-ratio coloring), PressureGauge, ConfBar, MixBar.

  • Pure inline SVG, responsive via 240px viewBox + preserveAspectRatio:none
  • ConfBar turns red below 50%; gauge needle signed −1…+1; VolBars color by ratio thresholds (≥1.6 accent, ≥2.5 red)
Depends: DS-1
DS-5Theming, density & settings3 pts

Runtime theme/accent/density/typeface switching with the transition-suppression guard; wire to a settings surface (prototype's Tweaks → production user settings).

  • Theme toggle in top bar swaps instantly with no color animation flash
  • Density changes reflow all spacing without layout breakage
  • Three typeface pairings select cleanly (sans+mono)
Depends: DS-1
E2

Data layer & tier engine

Phase 0
DL-1Ticker schema & API contract5 pts

Define the typed ticker object (see Data & Logic § Schema) and the endpoints that return it for single, portfolio, and scan contexts.

  • Schema typed end-to-end; A is nullable (single-ticker)
  • One endpoint per context returns the same shape; A scope differs (universe vs portfolio)
  • Tier-D / capped responses carry laneIssue, coverage, capNote
DL-2A/B/C indicator computation8 pts

Compute percentile (A), historical z-score (B), and raw metrics (C) per component from lane data.

  • B z-scores vs the name's own ≥20-session median per component
  • A percentile vs the active universe/portfolio
  • C ratios + focus share + pressure exactly per the C-field table
  • Insufficient history (<10 sessions) disables B for that component
Depends: DL-1
DL-3Direction & signing engine5 pts

Lee-Ready signing (quote rule → tick test; exclude midpoint/unknown), net signed pressure, signing confidence, and direction classification at the ±0.60 threshold.

  • Buyer/Seller require |pressure| ≥ 0.60; else Mixed/Undetermined
  • Signing confidence computed; <50% flagged "indicative only"
  • Direction derives only from flow — never price
Depends: DL-1
DL-4Rule-based tier classifier8 pts

Implement the A/B/C/D gates exactly as specified, mode-aware (skip A in single-ticker), returning per-rule pass/fail + gap + elevation eligibility for the Explain modal.

  • A/B/C gates match the Tier-classification section precisely
  • Provider-alert elevation logic correct; contextual flags never elevate
  • Classifier output drives Explain-tier with readable rule rows
  • Returns "why not next tier" gap reasons
Depends: DL-2, DL-3
DL-5Lane state contract5 pts

Model the 9 lane states with operator copy and tier effects; enforce suppression (D) and capping (C) rules.

  • All 9 states represented with the exact copy & dot colors
  • Required lane loading → Tier D; partial coverage <90% → cap at C
  • Optional/disabled lanes never penalize a tier
  • Coverage % surfaced; capNote banner shown when capped
Depends: DL-1
DL-6Corroboration evaluation3 pts

Evaluate the 6 flags across Strong/Moderate/Contextual independence and count independent confirmations for the A gate.

  • Strong = price / provider / options; ≥1 required for A
  • Panel shows count + restated elevation rule
Depends: DL-4

Build risks & guardrails

Where fidelity is easy to lose
  • Don't collapse A/B/C into a score. The instant a single "signal strength" number appears, the product's thesis is broken. Keep three lenses everywhere.
  • Direction is flow, not price. Any code path that sets direction from chg is a bug. Price is corroboration only.
  • Honest suppression. Tier D and capped-C must never show invented indicator values. Dashes and lane-health, not zeros.
  • Optional lanes are non-penalizing. Absent options/pre-market/alert lanes can only ever add corroboration, never subtract a tier.
  • Explain-tier must mirror the classifier. If the modal and the engine can disagree, trust is gone — derive the modal from the same evaluation.
